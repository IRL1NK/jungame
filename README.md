## Учебный проект: Консольный ASCII-Квест «Лабиринт»
Данный проект представляет собой учебную мини-игру с текстовым интерфейсом на языке C++. Разработка направлена на практическую демонстрацию принципов объектно-ориентированного программирования (ООП), декомпозицию задачи на логические модули и организацию управления динамической памятью.
## Цели проекта и проверяемые компетенции (2 курс)

* Архитектурное проектирование: разделение бизнес-логики, данных и интерфейса по отдельным классам.
* Управление памятью: инициализация объектов в куче (heap) через оператор new, работа с контейнерами указателей и деаллокация динамической памяти (delete) для предотвращения утечек.
* Взаимодействие классов: организация связей между независимыми объектами через передачу параметров и вызовы методов.
* Низкоуровневый ввод-вывод: обработка потока ввода в реальном времени без блокировки консоли.

## Архитектура проекта и структура классов
Программа разделена на заголовочные файлы (.h) и файлы реализации (.cpp). Архитектура включает в себя следующие компоненты:

   1. Класс Player (Игрок): отвечает за состояние персонажа в игровом мире. Хранит текущие декартовы координаты (X, Y) и целочисленный идентификатор комнаты, в которой находится игрок.
   2. Класс Location (Локация): инкапсулирует двумерную карту комнаты, представленную массивом строк (ASCII-символов). Отвечает за рендеринг геометрии уровня в консоль, проверку коллизий (доступности клеток для шага) и хранение статуса доступности локации для игрока.
   3. Класс Puzzle (Головоломка): отвечает за механику квестовых преград. Хранит строки вопроса и эталонного ответа, а также содержит метод валидации ввода пользователя.
   4. Модуль main.cpp (Игровой цикл): координирует работу всех классов. Реализует паттерн Game Loop: опрашивает клавиатуру, обновляет состояние игрока, проверяет триггеры переходов между локациями и условия победы, после чего вызывает метод перерисовки экрана.

## Игровой процесс и спецификация символов
Карта каждой комнаты состоит из статических ASCII-символов:

* # — стена (непроходимый объект).
* @ — текущая позиция игрока.
* (пробел) — свободная для перемещения клетка.
* 1, 2, 3 — двери-телепорты, ведущие в соответствующие локации.

Для прохождения игры необходимо переместиться из стартовой комнаты в финальную (№3). Доступ к промежуточной комнате заблокирован до тех пор, пока пользователь не введет правильный ответ на математическую задачу в модуле Puzzle.
## Управление
Ввод команд осуществляется в реальном времени с помощью функции _getch() из библиотеки <conio.h>. Это позволяет считывать нажатия клавиш мгновенно, без необходимости подтверждения через клавишу Enter.
Перемещение выполняется строго в английской раскладке клавиатуры:

* W — движение вверх (уменьшение координаты Y).
* S — движение вниз (увеличение координаты Y).
* A — движение влево (уменьшение координаты X).
* D — движение вправо (увеличение координаты X).

## Технические требования и системные зависимости

* Операционная система: Специфично для ОС Windows (используются вызовы system("cls") и заголовочный файл <conio.h>).
* Компилятор: Любой современный компилятор с поддержкой стандарта C++11 или выше (GCC/MinGW, MSVC).
* Локализация: Для корректного вывода кириллических символов в терминале Windows используется вызов setlocale(LC_ALL, "Russian").
